Arquitectura técnica — 2 enfoques

Elige cómo organizar el código

A

Monolito Electron + React

Todo en el mismo proceso: Electron con React en el renderer, gramjs (MTProto) directo desde el main process. Comunicación via IPC.

Pros

  • Más simple de construir
  • Un solo package.json
  • Arranque rápido

Contras

  • Menos separación de concerns
  • Pruebas más complejas
B

Electron + React + Backend Node separado Recomendado

Procesos separados: Electron (UI React) se comunica via HTTP/IPC con un backend Node que maneja la lógica de Telegram (gramjs), almacenamiento seguro de sesión y operaciones de archivos.

Pros

  • Separación clara UI / lógica Telegram
  • Backend testeable independientemente
  • Mejor seguridad (sesión aislada)
  • Fácil de testear con TDD

Contras

  • Más archivos/código boilerplate
  • Comunicación entre procesos
